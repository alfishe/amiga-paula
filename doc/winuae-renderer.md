# WinUAE Renderer Technical Documentation

## Overview

The WinUAE renderer implements Antti S. Lankila's audio emulation from WinUAE/UADE. It uses precomputed sinc tables that incorporate the filter response of each Amiga model, combining interpolation and filtering into a single convolution step.

## Algorithm

### Sinc Queue

Each voice maintains a circular queue of output state transitions:

```cpp
struct SincQueueItem {
    int time;    // Paula cycles since start
    int output;  // Delta from previous state
};
```

When the output state changes (sample × volume), the transition is recorded:

```cpp
if (v.sincOutputState != output) {
    v.sincQueueHead = (v.sincQueueHead - 1) & (SINC_QUEUE_LENGTH - 1);
    v.sincQueue[v.sincQueueHead].time = v.sincQueueTime;
    v.sincQueue[v.sincQueueHead].output = output - v.sincOutputState;
    v.sincOutputState = output;
}
```

### Sinc Interpolation

At each output sample, the current state is convolved with entries from the sinc integral table:

```cpp
int sum = v.sincOutputState << 17;

int offsetpos = v.sincQueueHead;
for (int j = 0; j < SINC_QUEUE_LENGTH; j++) {
    int age = v.sincQueueTime - v.sincQueue[offsetpos].time;
    if (age >= SINC_QUEUE_MAX_AGE || age < 0)
        break;
    sum -= winsinc[age] * v.sincQueue[offsetpos].output;
    offsetpos = (offsetpos + 1) & (SINC_QUEUE_LENGTH - 1);
}
```

The `winsinc_integral` tables contain precomputed windowed sinc integrals with the filter response baked in. This elegantly combines:
1. Band-limited interpolation (sinc)
2. Anti-aliasing
3. Amiga-specific analog filter modeling

### Sinc Tables

Five table variants exist in `sinctable.cpp`:

| Index | Model | LED Filter | Description |
|-------|-------|------------|-------------|
| 0 | A500 | Off | A500 static RC only |
| 1 | A500 | On | A500 + LED filter |
| 2 | A1200 | Off | Minimal (unused - see below) |
| 3 | A1200 | On | LED filter only |
| 4 | Vanilla | N/A | No filtering (flat response) |

For A1200 with LED off, the vanilla table is used since A1200 has no static filtering.

## Signal Chain

```
Paula DMA @ 3.55MHz → Sinc queue recording → Sinc interpolation → Analog filter (IIR) → Output
```

### Paula Cycle Simulation

The renderer tracks Paula cycles accurately:

```cpp
double nextAccum = cycleAccum + cyclesPerSample;
int cycles = static_cast<int>(nextAccum) - static_cast<int>(cycleAccum);

for (int cyc = 0; cyc < cycles; cyc++) {
    for (int ch = 0; ch < NUM_CHANNELS; ch++) {
        auto& v = voices[ch];
        if (v.periodCounter == 0) {
            v.period = v.storedPeriod;
            v.volume = v.storedVolume;
            v.periodCounter = v.period;
            fetchNextSample(v);
        }
        v.periodCounter--;
    }
}
```

### Analog Filters

After sinc interpolation, additional IIR filters model the analog stage:

**A500:**
```cpp
// Two cascaded RC sections
fs.rc1 = a500FilterA0_1 * input + (1.0f - a500FilterA0_1) * fs.rc1;
fs.rc2 = a500FilterA0_2 * fs.rc1 + (1.0f - a500FilterA0_2) * fs.rc2;

// Three cascaded RC sections for LED filter
fs.rc3 = ledFilterA0 * normalOutput + (1.0f - ledFilterA0) * fs.rc3;
fs.rc4 = ledFilterA0 * fs.rc3 + (1.0f - ledFilterA0) * fs.rc4;
fs.rc5 = ledFilterA0 * fs.rc4 + (1.0f - ledFilterA0) * fs.rc5;
```

**A1200:**
LED filter only (when enabled), no static filtering.

## Performance Characteristics

### Measured Results

Test module: tbl-tint_secondpart.mod @ 48kHz

| Metric | Value |
|--------|-------|
| MaxSlew | 34.0k |
| HF Energy | 1.07-1.24 |
| Character | Rich harmonics, softer transients |

### Comparison

- **vs BLEP**: Higher HF energy (1.24 vs 1.08), but softer transients (34k vs 37k slew)
- **vs PWM**: PWM with punch has both higher slew (50k) and similar HF energy (1.23)

WinUAE's sinc interpolation adds harmonic richness but the convolution inherently smooths transients. The PWM renderer achieves similar harmonic content while preserving sharper attacks.

## Implementation Notes

### Queue Management

The sinc queue uses a power-of-2 size for efficient modulo:

```cpp
static constexpr int SINC_QUEUE_LENGTH = 64;  // Must be power of 2
v.sincQueueHead = (v.sincQueueHead - 1) & (SINC_QUEUE_LENGTH - 1);
```

Entries older than `SINC_QUEUE_MAX_AGE` (2048 cycles) are ignored.

### Denormal Prevention

A small offset prevents FPU slowdown from denormalized floats:

```cpp
static constexpr float DENORMAL_OFFSET = 1e-10f;
fs.rc1 = a500FilterA0_1 * input + ... + DENORMAL_OFFSET;
```

### Table Generation

The sinc tables were generated by Antti S. Lankila using measured Amiga frequency responses. The tables encode the integral of the windowed sinc convolved with the filter impulse response, allowing efficient computation via subtraction.
